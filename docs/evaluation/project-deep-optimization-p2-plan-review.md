## 一致点 / 亮点

- 第一版直接消费后端已经落库的事实，避免新增前后端协议。
- 将“读了什么”和“为什么没读”放在用户消息附件区域，符合用户提问上下文的心智位置。
- 保持 assistant 过程面板职责不变，不把文件读取事实重复塞到多个入口。

## 偏差 / 不足

- 本计划暂不覆盖附件内容级截断原因；当前后端只提供可读/不可用与 token 估算。
- 不展示完整文件片段引用，后续若做 RAG 或引用来源，需要单独小步。
- 未覆盖浏览器视觉验证，当前以组件测试作为第一版验收。
