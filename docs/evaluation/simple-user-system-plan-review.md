## 一致点 / 亮点

- 保留聊天流式 v2 协议，不触碰核心 SSE 事件格式。
- 用 token 优先、`X-User-Id` 兼容兜底，能降低现有测试和灰度链路迁移风险。
- 不引入新依赖，避免在受限网络环境下阻塞实现。

## 偏差 / 不足

- 第一版 token 不能主动失效，登出只清理前端本地状态；这是简单系统可接受的边界。
- 未覆盖复杂账号安全策略，例如密码强度分级、登录失败锁定、刷新 token。
