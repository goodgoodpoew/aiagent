## 一致点 / 亮点

- 后端新增 `AuthModule`、`AuthService`、`AuthGuard`、`CurrentUser` 和 `Public`，没有破坏现有响应信封。
- `User` 增加认证字段并提供迁移，历史用户通过不可登录占位哈希平滑迁移。
- 聊天 v2、会话事件、文件上传和普通 request 都会携带认证信息。
- 文件下载从 `window.open` 改为带认证头的 fetch，避免下载接口因无法带 header 失效。
- 前端显式 `UMI_APP_USER_ID` 仍可作为灰度兼容入口，默认 demo 用户不再绕过登录页。

## 偏差 / 不足

- 全局 `AuthGuard` 当前负责解析 token，具体用户接口通过 `resolveUserId` 强制身份；这比“全局一刀切阻断所有无 token 请求”更兼容现有公开资源接口，偏差可接受。
- 未实现 refresh token、服务端登出撤销、登录失败锁定，这些在计划中属于非目标。
- 未运行真实数据库迁移、后端集成测试和浏览器 E2E；本次已用 unit/build 覆盖核心逻辑。
- `pnpm build` 会更新 `antdXStudy/src/.umi-production` 生成产物，属于构建噪声，交付时应避免把无关耗时数据当作源码变更评审。
