## 一致点 / 亮点

- 计划承认完整上下文压缩过早，只做必要的预算保护。
- 不改写 `Message` 原始记录，符合“数据库是真实历史”的原则。
- 将日志限定为计数和估算 token，避免泄露用户文本。
- 复用现有消息投影规则，减少协议偏差。

## 偏差 / 不足

- 默认 `maxPromptTokens=6000` 是经验值，不是模型能力声明；后续需要接入 provider/model 上下文窗口。
- 单条超长消息不做内容级截断，可能仍超过预算；本阶段通过“保留最近一条避免空 payload”接受该风险。
- 计划未覆盖真实上游模型请求验证，仅靠单元测试证明选择逻辑。
