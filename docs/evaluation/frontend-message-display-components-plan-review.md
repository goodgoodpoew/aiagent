## 一致点 / 亮点

- 没有把“未来任务、步骤、工具调用”一次性实现成复杂框架，只预留分组与组件位置。
- 保留 `Bubble.List` 作为 Ant Design X 生态入口，避免自建消息列表。
- 样式系统选择 token 到 CSS 变量的轻量方案，既能主题化又不过度封装。
- 明确不触碰后端协议、Redis、数据库和 `.umi`，没有踩项目核心约定。

## 偏差 / 不足

- 计划没有包含 Playwright 视觉验证；考虑本次主要是组件拆分，先用组件测试和 build 验证可以接受。
- `AnswerProcessPanel` 体量仍较大，计划仅要求迁移/复用，不强制拆成更多子组件；这符合“目前阶段够用”的原则。
