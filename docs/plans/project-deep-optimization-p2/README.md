# 项目深度优化 P2 拆分小单元计划索引

生成日期：2026-06-08

本目录把 [项目深度剖析与优先级优化计划](../../project-deep-optimization-priority-plan.md) 中的 P2 产品化与用户可解释性拆分为独立、可执行、可验收的小单元计划。

建议执行顺序：

1. [01 文件上下文可解释展示](./01-file-context-explainability-display.md)：让用户消息展示本轮实际进入模型上下文的附件、未进入上下文的附件原因和 token 估算。
2. 后续小步：失败重试/重新生成动作、取消生成动作、工作台入口收敛。

依赖关系：

```text
01 文件上下文可解释展示
 |
 +-- 后续失败恢复动作
 |
 +-- 后续工作台入口收敛
```

推进原则：

- 每步独立、可单独验收，依赖前序产物而非交织。
- 优先复用既有 `message-display` 组件、Redux 消息结构和 v2 协议，不新增前端状态机。
- 本阶段先展示后端已经持久化的事实，不在前端推断模型实际读取内容。
