# 移动端页面设计规范（微信小程序）

> 参考 **Vant Weapp**、**TDesign 小程序**、**uni-ui** 等常用组件库的页面结构与约束。  
> 色彩与项目内 `test.pen` 暖色主题令牌对齐；语义层与桌面端保持一致，便于品牌统一。  
> 桌面端规范见 [页面设计规范-桌面端.md](./页面设计规范-桌面端.md)。

---

## 参考标准与行业共识

本规范在组件库实践之上，与下列**广泛采用**的平台与无障碍标准对齐；数值冲突时以**更严格者**为准（小程序运行在 iOS / Android 宿主内，触控与字号宜贴近系统习惯）。

| 来源 | 要点摘要 | 说明 |
|------|----------|------|
| [Apple Human Interface Guidelines](https://developer.apple.com/design/human-interface-guidelines/)（HIG） | 触控目标建议不小于 **44×44 pt**；强调清晰层级、反馈与可达性 | iPhone 上微信小程序与系统控件同屏，宜与 44pt 热区一致 |
| [Material Design](https://m3.material.io/) | 触控目标最小 **48×48 dp**；布局常用 **4dp / 8dp** 步进与间距 | Android 宿主、部分设计团队以 48dp 为基准；图标之间常保留 **8dp** 间距以降低误触 |
| [WCAG 2.2](https://www.w3.org/TR/WCAG22/) | **对比度**：正文至少 **4.5:1**（AA），大字/粗体大字至少 **3:1**；**2.5.8 目标尺寸（最小）(AA)**：指针目标至少 **24×24 CSS 像素**，或满足等效间距例外；**2.5.5 目标尺寸 (AAA)**：建议 **44×44 CSS 像素** | Web 视图与小程序内嵌页可对照；本项目触控尺寸取 **≥44px 等效** 以同时贴近 HIG 与更高无障碍期望 |
| [微信小程序设计](https://developers.weixin.qq.com/miniprogram/design/) | 官方设计指引与审核相关约束 | 导航栏、授权、隐私等需符合平台规则 |

**单位对照（750 宽设计稿）**：`1px` 逻辑像素在标准宽度下约对应 `2rpx`，故 **44×44 pt ≈ 88×88 rpx**，**48×48 dp ≈ 96×96 rpx**（具体以实现端换算为准）。

## 1. 设计原则

| 原则 | 说明 |
|------|------|
| **清晰优先** | 卡片、列表为高频场景；信息层级用字重、字号、中性色区分，少用大面积强色块。 |
| **可预期** | 主操作在页面底部吸底；危险操作需二次确认；禁用态明确。 |
| **高效** | 下拉刷新 / 上拉加载；手势操作；减少用户操作步骤。 |
| **一致** | 全站同一套间距、圆角、字阶；与桌面端语义色、圆角档位、主色对齐，仅尺寸单位与导航形态按端裁剪。 |
| **轻量** | 每屏聚焦核心信息，避免内容堆砌，降低认知负担。 |
| **容错** | 操作结果 Toast 提示；关键操作可撤回或有明确提示。 |
| **可访问** | 对比度、触控热区、非仅依赖颜色传达状态；对齐 WCAG AA 与平台 HIG 的底线要求（见第 11 节）。 |

## 2. 色彩体系

### 2.1 语义令牌（与桌面端统一）

| 语义 | 用途 | 建议变量（与 `test.pen` 对齐） |
|------|------|--------------------------------|
| 主色 | 主按钮、链接、关键选中 | `$brand-primary` |
| 主色悬停/按下 | 按钮按下反馈 | `$brand-primary-active` |
| 主色浅底 | 选中背景、标签浅底 | `$brand-primary-muted` |
| 成功 | 成功状态、完成标签 | `$success` + `$success-bg` |
| 警告 | 警示、审核中 | `$warning` + `$warning-bg` |
| 危险 | 删除、报错、过期 | `$danger` + `$danger-bg` |
| 信息 | 提示、通知 | `$info` + `$info-bg` |
| 页面背景 | 大背景底色 | `$surface-subtle` 或 `$neutral-50` |
| 卡片/内容区背景 | 卡片、列表容器 | `$surface-default`（通常 `#FFFFFF`） |
| 边框 | 输入框边框 | `$border-default`；强调用 `$border-strong` |
| 分割线 | 列表分割、区块分割 | `$border-light` |
| 主文字 | 标题、正文 | `$text-primary` |
| 次要文字 | 说明文字 | `$text-secondary` |
| 辅助文字 | 占位、提示 | `$text-placeholder` |
| 禁用文字 | 禁用态 | `$text-disabled` |
| 遮罩 | 弹窗/弹出层背景 | `$overlay-scrim`（建议 `rgba(0,0,0,0.45)`） |

### 2.2 移动端专项注意

| 项目 | 说明 |
|------|------|
| 导航栏背景 | 默认品牌主色或白底 + 品牌标题色；需符合微信审核规范 |
| 状态栏文字 | 深色导航栏 → 白色文字；浅色导航栏 → 黑色文字 |
| 安全区着色 | 底部安全区域颜色与相邻元素一致，避免色差割裂 |
| 暗色模式 | 响应系统 `prefers-color-scheme: dark`；需额外适配 |

### 2.3 暗色模式（可选）

| 语义 | 亮色值 | 暗色值 |
|------|--------|--------|
| `$surface-default` | `#FFFFFF` | `#1F1F1F` |
| `$surface-subtle` | `#F5F5F5` | `#141414` |
| `$text-primary` | `rgba(0,0,0,0.88)` | `rgba(255,255,255,0.88)` |
| `$border-default` | `#D9D9D9` | `#424242` |

## 3. 字体与字阶

### 3.1 字体栈

- 使用 **系统字体**，不引入 webfont（减小包体积、提升渲染速度）。
- 金额数字可用 `font-variant-numeric: tabular-nums`（若基础库支持）。

### 3.2 字阶

| Token | 字号 (rpx) | 行高 | 字重 | 典型用途 |
|-------|-----------|------|------|----------|
| `text-display` | 40–44 | 1.35 | 600 | 页面大标题（少用，一般由导航栏标题承担） |
| `text-title` | 34–36 | 1.4 | 600 | 卡片标题、弹窗标题 |
| `text-subtitle` | 30–32 | 1.5 | 500–600 | 二级区块标题 |
| `text-body` | 28–30 | 1.5–1.6 | 400 | 正文、主要内容 |
| `text-body-secondary` | 26–28 | 1.5 | 400 | 描述、说明（颜色用 `$text-secondary`） |
| `text-caption` | 24 | 1.5 | 400 | 时间戳、标签、辅助信息 |
| `text-small` | 22–24 | 1.5 | 400 | 脚注 |

- **最小字号**：正文不低于 **24rpx**（约 12px）。
- 正文行高 **1.5–1.6 倍**；标题行高 **1.3–1.4 倍**。

## 4. 圆角、描边与阴影

### 4.1 圆角档位

| Token | 值 (rpx) | 用途 |
|-------|---------|------|
| `radius-none` | 0 | 特殊场景 |
| `radius-sm` | 8 | 输入框、小标签 |
| `radius-md` | 12–16 | 按钮、卡片 |
| `radius-lg` | 24 | 大卡片、空状态容器 |
| `radius-xl` | 32 | 底部弹层顶部圆角 |
| `radius-full` | 9999px | 胶囊标签、头像 |

### 4.2 描边

- 默认边框：`1px` + `$border-default`。
- 输入框通常用底线样式；focus 时底线变为品牌色。

### 4.3 阴影与分层

- **少用重阴影**，以 **白底卡片 + 灰色页面背景** 自然分层为主。
- 必要时使用轻阴影：`0 2rpx 8rpx rgba(0,0,0,0.05)`。
- 弹出层可用：`0 -4rpx 16rpx rgba(0,0,0,0.08)`。

## 5. 间距与尺寸

### 5.1 基础单位

- 以 **750rpx** 宽设计稿为基准（对应 375px 物理逻辑宽度）。
- 间距以 **16rpx** 为基本步进（兼顾 **8rpx** 微调）；与 **Material 8dp 栅格**、**Apple 常见 8pt 增量**可对照，便于跨端设计系统统一。
- **常用间距**：`8 / 16 / 24 / 32 / 40 / 48rpx`。
- **相邻可点击元素**：除满足各自最小热区外，视觉或布局上宜保留 **≥8rpx（约 4px）** 级间距，避免密集误触；重要并列按钮之间可加大至 **16rpx 及以上**。

### 5.2 页面与容器

| 场景 | 数值 |
|------|------|
| 页面左右边距 | 32rpx |
| 卡片内边距 | 24–32rpx |
| 卡片之间间距 | 24rpx |
| 列表项分割线缩进 | 左侧 32rpx（对齐文字） |
| 模块之间间距 | 24rpx |
| 表单项垂直间距 | 以分割线/分组分隔 |
| 按钮距屏幕底部 | ≥ `safe-area-inset-bottom` + 16rpx |

### 5.3 触摸热区

**行业对照**：Apple HIG 建议最小约 **44×44 pt**；Material Design 建议最小 **48×48 dp**。[微信小程序设计指南](https://developers.weixin.qq.com/miniprogram/design/) 说明：在手机上点击控件的**物理尺寸**换算后大致在 **7mm–9mm** 区间，与上述 pt/dp 规格一致，并推荐参考或沿用微信标准组件尺寸。小程序在 iOS 上可优先对齐 **44pt 等效（88rpx）**；若产品统一按 Android 主规格验收，可将关键控件提升至 **48dp 等效（96rpx）**。视觉较小的图标按钮须通过 **向外扩展热区（padding）** 达到最小尺寸，而非仅缩小点击区域。

| 元素 | 最小尺寸 |
|------|---------|
| 按钮 | 高度 ≥ 80rpx（大按钮 **88rpx**，与 44pt 等效一致） |
| 列表行 | 高度 ≥ 88–96rpx（单行含分割线） |
| 图标按钮 | 热区 ≥ **88×88rpx**（**44×44px**）；对齐 Material 48dp 时用 **96×96rpx** |
| Tab 栏单项 | 各项等分，高度 ≥ 98rpx |
| 搜索框 | 高度 ≥ 72rpx |

### 5.4 安全区

- **顶部**：自定义导航栏需预留 `statusBarHeight + navBarHeight`（通常 44px）。
- **底部**：底部固定按钮 / TabBar 需加 `safe-area-inset-bottom` 内边距。
- **刘海屏**：左右无需额外适配（微信已处理）。

## 6. 布局架构

### 6.1 页面结构

```
┌──────────────────────┐
│ 状态栏 (系统)          │
├──────────────────────┤
│ 导航栏：← 返回  标题    │  44px
├──────────────────────┤
│                      │
│  页面滚动内容区        │
│  ┌────────────────┐  │
│  │ 搜索栏（可选）   │  │
│  ├────────────────┤  │
│  │ 筛选 Tab（可选） │  │
│  ├────────────────┤  │
│  │ 列表/卡片/表单   │  │
│  │ ...             │  │
│  │ 加载更多/空状态  │  │
│  └────────────────┘  │
│                      │
├──────────────────────┤
│ 底部操作栏（可选）     │
│ safe-area-bottom     │
└──────────────────────┘
```

### 6.2 Tab 页结构

```
┌──────────────────────┐
│ 导航栏               │
├──────────────────────┤
│                      │
│  各 Tab 对应页面内容   │
│                      │
├──────────────────────┤
│ 首页   列表   我的     │  ← TabBar（图标+文字）
│ safe-area-bottom     │
└──────────────────────┘
```

### 6.3 导航规范

| 模式 | 说明 |
|------|------|
| 原生导航栏 | 标题居中；自动带返回箭头；**推荐优先使用** |
| 自定义导航栏 | 需手动处理状态栏高度、胶囊按钮避让、返回逻辑 |
| TabBar | 2–5 个 Tab；图标 + 文字；选中色 `$brand-primary` |
| 分段控制 | 页内二级切换；不超过 5 项 |

## 7. 图标规范

### 7.1 图标风格

| 规范项 | 要求 |
|--------|------|
| 风格类型 | 与桌面端统一，全站 **线形（Outlined）** 或 **填充（Filled）** |
| 线宽 | **1.5–2px**，全套图标保持一致 |
| 端点/拐角 | 统一使用 **圆角端点（Round Cap/Join）** 或 **直角端点**，不混搭 |
| 视觉重心 | 图标在画布内居中对齐，视觉重心一致 |
| 色彩 | 默认继承父元素文字颜色（`currentColor`），不单独着色 |

### 7.2 图标尺寸体系

| Token | 设计稿尺寸 (rpx) | 实际尺寸 (px, 375屏) | 适用场景 |
|-------|-----------------|---------------------|---------|
| `icon-xs` | 24rpx | 12px | 极小指示器、状态点旁 |
| `icon-sm` | 32rpx | 16px | 列表右侧箭头、输入框后缀 |
| `icon-md` | 40rpx | 20px | 按钮前缀图标、Cell 左侧图标、分段控制图标 |
| `icon-lg` | 44rpx | 22px | TabBar 图标、导航栏操作图标 |
| `icon-xl` | 56–64rpx | 28–32px | 宫格入口图标、功能入口 |
| `icon-xxl` | 80–96rpx | 40–48px | 空状态辅助图标、引导页图标 |
| `icon-hero` | 120–160rpx | 60–80px | 空状态主图标、结果页主图标 |

### 7.3 图标与文字搭配

| 场景 | 图标尺寸 | 间距（图标与文字） | 对齐方式 |
|------|---------|-----------------|---------|
| 大按钮（88rpx） | 36–40rpx | 8–12rpx | 垂直居中 |
| 中按钮（72rpx） | 32–36rpx | 8rpx | 垂直居中 |
| 小按钮（56rpx） | 28–32rpx | 4–8rpx | 垂直居中 |
| Cell 列表项（左图标） | 40–44rpx | 16–24rpx | 垂直居中 |
| Cell 列表项（右箭头） | 32rpx | 8rpx | 垂直居中 |
| 分段控制 Tab | 32–36rpx | 8rpx | 垂直居中 |
| Toast 提示 | 48–56rpx | 16rpx（图标在上，文字在下） | 水平居中 |
| NoticeBar | 32–36rpx | 8rpx | 垂直居中 |

### 7.4 图标交互态

| 状态 | 表现 |
|------|------|
| 默认 | `$text-secondary` 或 `$text-primary`（根据层级） |
| 按下（active） | 颜色加深 或 整区域 opacity 0.7 |
| 禁用 | `$text-disabled` |
| 选中 | `$brand-primary`；可切换为填充变体 |
| 未选中 | `$text-placeholder` 或 `$text-secondary` |

### 7.5 TabBar 图标

| 规范项 | 要求 |
|--------|------|
| 尺寸 | **44–48rpx**（22–24px），各 Tab 图标尺寸统一 |
| 双态设计 | **未选中**：线形 + `$text-placeholder`；**选中**：填充 + `$brand-primary` |
| 图标下方文字 | 20–22rpx，间距 4rpx |
| 画布规格 | 建议 **48×48px**（导出 @2x 为 96×96px，@3x 为 144×144px） |
| 格式 | PNG（@2x + @3x）或 SVG；微信原生 TabBar 需 PNG |
| Badge | 右上角红点或数字角标，不遮挡图标主体 |
| 动效 | 切换时可加微动效（缩放、弹跳），≤ 200ms |

### 7.6 各场景图标用法

#### 导航栏

| 元素 | 图标 | 尺寸 |
|------|------|------|
| 返回 | `arrow-left` / `chevron-left` | 40–44rpx |
| 首页 | `home` | 40rpx |
| 更多 | `ellipsis` / `more-horizontal` | 40rpx |
| 分享 | `share` | 40rpx |
| 关闭 | `close` | 40rpx |

#### 列表 Cell

| 元素 | 图标 | 尺寸 |
|------|------|------|
| 左侧类型图标 | 语义图标（如 `file`、`user`） | 40–44rpx |
| 右侧箭头 | `chevron-right` | 28–32rpx，`$text-placeholder` |
| 右侧开关 | Switch 组件（非图标） | — |
| 选中勾 | `check` | 36rpx，`$brand-primary` |

#### 表单

| 元素 | 图标 | 尺寸 |
|------|------|------|
| 输入框清除 | `close-circle` / `clear` | 32rpx，`$text-placeholder` |
| 密码显隐 | `eye` / `eye-off` | 36rpx |
| 下拉箭头 | `chevron-down` | 28rpx |
| 日期选择 | `calendar` | 36rpx |
| 图片上传 | `plus` 或 `camera` | 48rpx，`$text-placeholder` |
| 图片删除 | `close-circle-filled` | 32rpx，右上角悬浮 |
| 必填标记 | 红色 `*`（文字，非图标） | — |

#### 反馈与状态

| 元素 | 图标 | 尺寸 | 颜色 |
|------|------|------|------|
| Toast 成功 | `check-circle` / `check` | 48–56rpx | `$success` 或白色 |
| Toast 失败 | `close-circle` / `close` | 48–56rpx | `$danger` 或白色 |
| Toast 加载 | `loading` 旋转 | 48–56rpx | 白色 |
| Dialog 警告 | `exclamation-circle` | 64rpx | `$warning` |
| Empty 空数据 | 自定义插图 | 120–160rpx | `$text-disabled` 色系 |
| NoticeBar | `volume` / `info-circle` | 32rpx | 随语义色 |
| 标签状态 | `check-circle`（成功）/ `clock`（待审）/ `close-circle`（失败） | 24–28rpx | 随语义色 |

#### 功能入口宫格

| 规范项 | 要求 |
|--------|------|
| 图标尺寸 | 56–64rpx |
| 图标样式 | 可用品牌色渐变背景 + 白色图标，或纯图标 |
| 背景容器 | 圆角方形（`radius-lg`）或圆形（`radius-full`） |
| 背景颜色 | 对应功能语义浅色（如订单用 `$brand-primary-muted`） |
| 文字 | 图标下方 24–26rpx，间距 8–12rpx |
| 宫格列数 | 4 列或 5 列，行间距 24–32rpx |

### 7.7 图标库选型建议

| 图标库 | 图标数量 | 风格 | 推荐场景 |
|--------|---------|------|---------|
| **Vant Icons** | 250+ | 线形/填充 | 配合 Vant Weapp |
| **TDesign Icons** | 1000+ | 线形/填充 | 配合 TDesign 小程序 |
| **uni-icons** | 100+ | 填充 | 配合 uni-ui（图标较少，需补充） |
| **Remix Icon** | 2500+ | 线形/填充 | 丰富图标需求 |
| **IconPark (ByteDance)** | 2000+ | 4 种主题 | 多风格需求 |

**选型原则**：

1. **优先使用组件库内置图标**（Vant Icons / TDesign Icons），减少额外依赖。
2. 内置图标不够用时从同风格第三方库补充，线宽与圆角需一致。
3. 自定义图标严格遵循主库的画布尺寸、线宽、安全区规范。
4. 小程序注意包体积：使用字体图标（iconfont）或按需引入 SVG。

### 7.8 图标资源管理

| 规范项 | 要求 |
|--------|------|
| 格式 | **字体图标**（iconfont，推荐，体积小）或 **SVG**（灵活，需处理兼容） |
| TabBar 图标 | PNG 格式，需提供 @2x（81×81px）和 @3x（适配高分屏） |
| 命名规则 | 小写短横线连接：`icon-{类别}-{名称}`，如 `icon-nav-home`、`icon-action-delete` |
| iconfont 使用 | 上传至 iconfont.cn 或内部平台；下载为小程序组件格式 |
| 分类 | 按用途分：`navigation/`、`action/`、`status/`、`tabbar/` 等 |
| 组件封装 | 封装 `<icon>` 组件，支持 `name`、`size`、`color` 属性 |
| 包体积控制 | 仅打包项目实际使用的图标，避免全量引入 |
| 设计交付 | Figma/Sketch 中图标以 Component 管理，标注尺寸与颜色 |

### 7.9 图标设计规范（自定义图标时）

| 规范项 | 要求 |
|--------|------|
| 画布 | 标准 **48×48px**（@2x 视网膜基准） |
| 安全区 | 内容区域留 **4px** 内边距（即内容区 40×40px） |
| 线宽 | **3px**（@2x 下，对应 1.5px @1x） 或 **4px**（对应 2px @1x） |
| 圆角 | 线条拐角与端点统一风格 |
| 对齐 | 关键点对齐 **像素网格**，避免半像素模糊 |
| 颜色 | 使用单色，导出时不写死色值 |
| 视觉校正 | 圆形/三角形图标适当放大（约 102–104%），使其视觉等大 |
| 导出 | TabBar：PNG @2x + @3x；其他：SVG 或字体图标 |

## 8. 插图与空状态

| 属性 | 规范 |
|------|------|
| 风格 | 线性 或 轻渐变，与图标风格一致 |
| 尺寸 | 空状态插图宽度 **200–320rpx** |
| 颜色 | `$neutral-300` ~ `$neutral-500` 为主色调，可加入品牌色点缀 |
| 场景 | 至少三套：空数据、无网络、无权限 |
| 文案结构 | 插图 → 主文案（30rpx，`$text-primary`）→ 次文案（26rpx，`$text-secondary`）→ 操作按钮（可选） |
| 居中 | 在可视区域内水平垂直居中 |
| 底部留白 | 空状态区域距页面底部留足空间，不被吸底按钮遮挡 |

## 9. 组件规范

### 9.1 按钮

| 类型 | 用途 |
|------|------|
| Primary | 主操作（提交、确认、下一步） |
| Default | 次要操作（取消、重置） |
| Plain（幽灵） | 轻量操作、次要入口 |
| Danger | 删除、解绑 |

| 尺寸 | 高度 | 场景 |
|------|------|------|
| 大 | 88–96rpx | 页面底部主操作（通栏或左右留 32rpx） |
| 中 | 72–80rpx | 卡片内操作 |
| 小 | 56–64rpx | 行内操作、标签按钮 |

- **吸底按钮**：固定页面底部，必须加 `safe-area-inset-bottom`。
- **加载态**：显示 loading + 禁用，防止重复提交。
- **双按钮**：主按钮在右，次按钮在左。
- **图标按钮**：仅图标时热区 ≥ 88×88rpx。

### 9.2 列表 Cell

| 规范项 | 说明 |
|--------|------|
| 结构 | 左图标/头像（可选）+ 主标题 + 描述（可选）+ 右侧值/箭头 |
| 高度 | 单行 88–96rpx；双行 120–140rpx |
| 分割线 | 左缩进与文字对齐 |
| 分组标题 | 灰色 24rpx，内边距对齐列表 |
| 点击反馈 | 背景变灰（`hover-class`），150ms |
| 滑动操作 | 仅必要时使用，需配合确认 |

### 9.3 表单

| 规范项 | 说明 |
|--------|------|
| 布局 | 左标签 + 右输入 或 上标签 + 下输入 |
| 选择器 | 原生 Picker / 自定义弹层；注意 z-index 与滚动穿透 |
| 键盘类型 | 金额 `digit`，手机号 `number`，身份证 `idcard` |
| 必填标记 | 红色 `*` 在标签前 |
| 校验 | 失焦 + 提交；错误时底线变红 + 文字提示 |
| 提交按钮 | 吸底，加 safe-area |
| 长表单 | 分步 Step 或分组展示 |

### 9.4 反馈

| 类型 | 组件 | 规范 |
|------|------|------|
| 轻量提示 | Toast | 居中短暂显示；文案 ≤ 7 汉字 |
| 重要确认 | Dialog | 标题 + 内容 + 双按钮；主操作在右 |
| 操作集合 | ActionSheet | 底部弹出；最多 6 项 + 取消 |
| 全局通知 | NoticeBar | 顶部横幅；可滚动；可关闭 |
| 加载 | Loading / Skeleton | 首屏骨架屏；列表底部 loading |
| 下拉刷新 | PullRefresh | 列表页标配 |
| 空状态 | Empty | 居中插图 + 文案 + 按钮 |

### 9.5 弹出层

| 组件 | 方向 | 场景 |
|------|------|------|
| Popup（底部） | 从下往上 | 筛选面板、选择器、操作确认 |
| Popup（右侧） | 从右往左 | 详情面板、侧滑筛选 |
| Popup（居中） | 居中 | Dialog、轻量编辑 |
| Popup（顶部） | 从上往下 | 下拉通知 |

- 弹出层圆角：底部弹出顶部圆角 `radius-xl`，顶部弹出底部圆角 `radius-xl`。
- 遮罩点击默认关闭非破坏性弹层。

### 9.6 搜索

| 规范项 | 说明 |
|--------|------|
| 位置 | 页面顶部，导航栏下方 |
| 样式 | 圆角搜索框，左侧搜索图标，右侧清除按钮 |
| 交互 | 点击跳转搜索页 或 展开搜索；支持历史记录与热门推荐 |
| Debounce | 300ms |

## 11. 无障碍

本节与 [WCAG 2.2](https://www.w3.org/TR/WCAG22/)、[Apple 辅助功能](https://developer.apple.com/design/human-interface-guidelines/accessibility) 等要求对齐；合规项目请按法务/产品等级（**AA / AAA**）选用指标。

| 项目 | 要求 |
|------|------|
| 对比度 | **文本**（AA **1.4.3**）：正文与背景 ≥ **4.5:1**；**大号字**（如约 ≥18pt 常规或 ≥14pt 粗体）≥ **3:1**。**非文本**（AA **1.4.11**）：用户界面组件、图形及其相邻色的对比至少 **3:1**（需单独核对图标、描边、焦点环等） |
| 触摸热区 | 可点击区域 ≥ **44×44px**（**88×88rpx**），与 Apple HIG 一致；若需对齐 Material **48×48dp**，采用 **96×96rpx**。WCAG **2.5.8 目标尺寸（最小）(AA)** 规定至少 **24×24 CSS 像素**或符合间距例外；追求更高包容性时可对照 **2.5.5 目标尺寸 (AAA)** 的 **44×44 CSS 像素** |
| 相邻目标 | 过小的并列控件应保证 **不重叠**的可达点击区域，或满足 WCAG 2.5.8 的等效间距说明 |
| 文案 | 按钮用动词；错误信息说明如何修复 |
| 色盲友好 | 状态不仅靠颜色，辅以图标或文字（WCAG **1.4.1 颜色使用**） |
| 动效 | 尊重系统 **减弱动态效果**（`prefers-reduced-motion`）；避免仅依赖闪烁传达关键信息（**2.3.1 三段闪烁或低于阈值**） |
| 焦点与朗读 | 页面结构语义清晰；图标按钮需可访问名称（小程序可用 `aria-label` 等能力，以基础库文档为准；能力不足时用可见文案兜底） |
| 国际化 | 预留文案长度 +30%；日期时间格式可配置 |

## 12. 性能与体验

| 项目 | 建议 |
|------|------|
| 首屏渲染 | 使用骨架屏（Skeleton），避免长时间白屏 |
| 图片加载 | 懒加载（`lazy-load`）+ loading 占位 |
| 列表性能 | 长列表用虚拟滚动（`recycle-view`）或分页加载 |
| 分包加载 | 非核心页面放入子包（主包 ≤ 2MB） |
| 预加载 | `preloadRule` 预加载可能访问的子包 |
| 动画 | CSS 动画或 WXS；避免频繁 setData |
| setData 优化 | 减少数据量，避免大对象 |
| 图标体积 | 使用 iconfont 或按需引入，避免全量打包 |
| 缓存 | `wx.setStorageSync` 缓存低频变更数据 |
| 网络异常 | 全局处理；提供重试按钮；离线提示 |
| 页面栈 | 注意 10 层限制；适时用 `redirectTo` |
| 动效与可访问性 | 装饰性动画时长宜 **200–300ms** 量级；若用户开启系统减弱动效，应减少自动播放与大幅度位移动画（与第 11 节一致） |

## 14. 与 `test.pen` 的衔接建议

1. 在画布中增加「列表页线框」、「表单页」、「个人中心」Frame，仅用语义色与字阶占位。
