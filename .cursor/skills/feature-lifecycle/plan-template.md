# 拆分小步计划模板

用于 `docs/plans/<feature>/NN-<step>.md`。每一步必须完整包含全部字段，缺一不可。把 `<...>` 占位替换为实际内容；`实施方案 / 产出 / 验收` 建议用 `- [ ]` 勾选项，执行时改为 `- [x]`。

> 拆分硬性原则：每一步实现必须独立，即使互相引用也要限定在可控范围内（一步只处理一个清晰关注点，依赖前序步骤的产物，而不是与之交织）。

---

```markdown
# NN <步骤标题>

> 执行标注（YYYY-MM-DD）：<完成情况 / 是否已 build/test 通过 / 哪些未实地验证>

## 修改位置

新增：
- [ ] `<path/to/new-file>`（用途）

修改：
- [ ] `<path/to/file>`（改动点）

## 目的

<这一步要达成的直接结果，一句话可判定。>

## 动机

<为什么现在做这一步；它在整体功能中的位置与价值。>

## 修改原因

- <现状的具体问题1（耦合 / 缺失 / 重复 / 风险）>
- <现状的具体问题2>

## 实施方案

1. [ ] <可执行步骤1>
2. [ ] <可执行步骤2>
3. [ ] <可执行步骤3>

（必要时附关键接口/类型/伪代码片段，限定在本步范围内）

## 产出

- [ ] <可观察的交付物1，如新增端点 / 新增服务 / 新增事件>
- [ ] <可观察的交付物2>

## 验收

- [ ] <可验证的标准1（行为 / 测试 / 构建）>
- [ ] <可验证的标准2>
- [ ] <不破坏的既有能力，如不影响旧端点 / 旧链路>

## 风险与注意事项

- <边界约束：本步不要做什么，避免越界到下一步>
- <潜在风险与回滚 / 兜底方式>
- <需遵守的项目约定：v2 协议、中文注释、不改 src/.umi、不提交 .env 等>
```

## 索引文件（README）建议结构

`docs/plans/<feature>/README.md`：

```markdown
# <功能> 拆分小单元计划索引

生成日期：YYYY-MM-DD

本目录把 [<总计划>](../<feature>-plan.md) 拆分为独立、可执行、可验收的小单元计划。

建议执行顺序：
1. [01-...](./01-....md)：<一句话>
2. [02-...](./02-....md)：<一句话>

依赖关系：

\`\`\`text
01
 |
 +-- 02 -- 03
\`\`\`

推进原则：
- 每步独立、可单独验收，依赖前序产物而非交织。
- 优先复用既有分层与约定，不另起炉灶。
```
